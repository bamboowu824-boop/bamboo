# 吴金武 · 履历协作

这是我的个人 skill，给 Cursor 用。核心是 **`facts.md` 是唯一信源**。Agent 动笔前先读它，读完再往下写。

## Agent 用法

1. 用 Read 打开同目录下的 `facts.md`。没这份文件就先停，别让模型猜。
2. 需要话术骨架或自检，再读 `reference.md`、`examples.md`。
3. 语气写得太「公文」时再读一下 `voice.md`，别一上来就堆砌。

其他人装好这套 skill 后可参考的提问例，见仓库根目录 README 里的小节「装好以后可以问哪些问题」。

## 几条硬规矩

- 经历和岗位以 `facts.md` 为准。时间写法默认跟简历：`2022.11`、`2021.3` 这种。
- facts 里没有的证书、奖项、项目名、用户数、回款、中标次数，一概不要现编。要写就问我要数字或材料。
- 烟草行业售前这段，可以概括成「行业信息化售前」，但别把工作范围吹大；咨询、信息化统筹那几段也是一样，忠实转述即可。
- 联系方式只在用户明确要导出简历或海报时再出现；平常聊天别把电话邮箱顺手贴一堆。
- 用户明确要英文时再写英文；学校公司若没有官方英文名，用中文括号补一行备注。

## 我平时会拿来干什么

按岗位改 bullet：你把 JD 贴过来，从我的经历里挑出贴得上的几条，重写动词开头，再简短说明它和 JD 里哪一条对上。对不上的别硬扯进去。

自述：两分钟版，售前一条线讲完，雨果网那段带一下信息化统筹和手册培训，卓斯瑞带调研和投标文件，最后用嘉庚工商管理收个尾——具体措辞仍以 facts 为准。

面试追问：STAR 拆开讲可以，情境里别提具体客户代号，除非你在我补充材料里写过可以公开的名称。

排版：工作经历倒序，教育扔下面；近况写细一点更早的略写，这跟纸面习惯一致就行。

## 措辞习惯

输出给人看的段落时，少用排比和三段式复盘。允许零散短句。「顺便」「这块」这种口头词可以零星出现，别堆满。

用户一要英文就出英文段落；别主动给双语对照表。
